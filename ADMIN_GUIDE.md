# Инструкция по админке GEOweb

Эта инструкция описывает все основные разделы админки, формы и поля.

Админка открывается по адресу:

```text
http://localhost:3000/admin
```

Если сайт запущен на сервере, вместо `localhost:3000` будет домен сайта.

## 1. Общая логика админки

Админка управляет содержимым публичного сайта:

- учебными материалами;
- публикациями;
- фотографиями;
- научными темами;
- архивными материалами;
- библиотекой чужих работ;
- картой экспедиций;
- сообщениями с формы контактов;
- файлами, загруженными на сайт;
- текстами главной страницы и страницы "Обо мне".

Данные сайта хранятся в локальной базе SQLite в папке `data`. Загруженные файлы хранятся в папке `public/uploads`.

## 2. Верхняя панель

В верхней панели находятся:

- название сайта;
- переключатель режима "Публичный сайт / Админка";
- переключатель темы, если он включен;
- индикатор сохранения.

В режиме добавления материалов индикатор может показывать, что редактируется черновик или новый материал.

## 3. Левое меню админки

Слева находится меню разделов:

```text
Дашборд
Учебные материалы
Публикации
Фотографии
Научные темы
Архивные материалы
Библиотека
Карта экспедиций
Сообщения
Файловый менеджер
Страницы сайта
```

Кнопка внизу меню ведет на быстрое добавление нового материала.

## 4. Дашборд

Раздел:

```text
/admin
```

Это главная панель управления.

На дашборде видно:

- общее количество материалов;
- количество учебных материалов;
- количество публикаций;
- количество фотографий;
- количество научных тем;
- количество архивных материалов;
- количество черновиков;
- недавно добавленные учебные материалы;
- шаблоны быстрого создания;
- посещаемость сайта;
- резервная копия.

### Карточки статистики

Карточки показывают количество записей в каждом разделе. По карточкам можно кликать, чтобы перейти в соответствующий раздел.

### Недавние материалы

Показывает последние учебные материалы. При клике открывается форма редактирования этого материала.

### Шаблоны

Шаблоны заранее заполняют часть полей в форме добавления.

Доступные шаблоны:

- лекция;
- презентация для семинара;
- статья в журнал.

После выбора шаблона откроется форма "Быстрое добавление", где поля уже частично заполнены.

### Посещаемость

Показывает просмотры публичных страниц:

- за 7 дней;
- за 30 дней;
- всего;
- график по дням;
- популярные страницы.

Статистика собирается самим сайтом, без внешних сервисов и cookie.

### Резервная копия

Карточка "Резервная копия" позволяет:

- скачать полную копию базы данных;
- восстановить базу данных из файла.

Важно: восстановление заменяет все текущие данные портала содержимым выбранного файла. Перед восстановлением текущая база автоматически сохраняется рядом как запасная копия.

## 5. Быстрое добавление

Раздел:

```text
/admin/add
```

Здесь создаются и редактируются основные сущности:

- учебный материал;
- публикация;
- фотография;
- научная тема;
- архивный документ.

Вверху формы есть переключатель вида:

- "С превью" — слева форма, справа предварительный вид карточки;
- "Фокус" — только форма, без превью.

Если создается новая запись, сверху доступны вкладки типа материала. Если редактируется существующая запись, тип уже выбран.

## 6. Общие элементы формы добавления

Эти элементы встречаются почти во всех формах.

### Загрузка файлов

Для учебных материалов, публикаций, фото и архивных документов сверху есть зона загрузки.

Можно:

- нажать на зону и выбрать файл;
- перетащить файл мышью;
- выбрать файл из уже загруженных.

При загрузке сайт спрашивает:

```text
В какую папку сохранить?
```

Можно выбрать одну из папок:

```text
photos
lectures
publications
archive
library
documents
```

или создать новую папку.

Файл будет сохранен в:

```text
public/uploads/имя-папки
```

На сайте путь будет выглядеть так:

```text
/uploads/имя-папки/файл.pdf
```

### Кнопка "Выбрать из загруженных"

Открывает файловый выборщик.

В нем есть две вкладки:

- "Загрузить новый" — загрузить файл с компьютера;
- "Из загруженных" — выбрать файл, который уже лежит в `uploads`.

### Загруженные приложения

Для учебных материалов, публикаций и архивных документов можно прикрепить несколько файлов.

Каждый прикрепленный файл отображается отдельной строкой. Крестик справа удаляет файл из списка приложений записи. Сам файл на диске при этом может остаться в файловом менеджере.

### Название

Главное название записи.

Для учебного материала это название карточки в разделе "Студентам".

Для публикации это название статьи, монографии или тезисов.

Для фото это подпись фотографии.

Для научной темы это название темы.

Для архивного документа это название архивной записи.

### Теги

Поле тегов работает так:

1. Введите тег.
2. Нажмите `Enter` или поставьте запятую.
3. Тег появится отдельной плашкой.

Крестик внутри плашки удаляет тег.

Ниже могут отображаться подсказки тегов. Нажатие на подсказку добавляет тег.

Теги используются для поиска, группировки и более понятного описания материала.

### Уровень доступа

Доступ определяет, кто увидит материал на публичном сайте.

Варианты:

```text
Открыто
```

Видно всем посетителям сайта.

```text
Только студентам
```

Материал помечается как студенческий. Подходит для учебных файлов, которые не надо показывать как полностью публичные.

```text
По ссылке
```

Материал не должен активно выводиться в обычных публичных списках, но может быть доступен по прямой ссылке или через связанные блоки, если такая логика используется.

```text
По запросу
```

Материал виден как запись, но доступ к файлу предполагается по запросу.

```text
Только владелец
```

Служебный или закрытый материал. Обычно не предназначен для публичного просмотра.

### Сохранить черновик

Кнопка сохраняет запись как черновик.

Для учебных материалов черновик не считается опубликованным материалом.

Для публикаций черновик сохраняется с закрытым доступом.

Используйте черновик, если запись еще не готова.

### Опубликовать

Кнопка сохраняет запись как опубликованную.

После публикации данные попадают на публичный сайт согласно выбранному уровню доступа.

## 7. Форма "Учебный материал"

Выбирается во вкладке:

```text
Учебный материал
```

Используется для лекций, презентаций, методичек, таблиц, заданий и других материалов для студентов.

### Файл или приложения

Можно загрузить один или несколько файлов.

Если основной файл еще не выбран, первый загруженный файл автоматически становится основным файлом материала.

Подходящие форматы:

- PDF;
- DOCX;
- PPTX;
- XLSX;
- ZIP;
- изображения;
- другие учебные файлы.

### Поля формы

```text
Название
```

Название учебного материала.

Пример:

```text
Введение в палеонтологию беспозвоночных
```

```text
Дисциплина
```

Курс или дисциплина, к которой относится материал.

Можно выбрать из подсказок или ввести новую дисциплину.

Примеры:

```text
Палеонтология
Стратиграфия
Кристаллография
Полевая практика
```

```text
Тип материала
```

Тип учебного ресурса.

Можно выбрать из подсказок или ввести свой тип.

Примеры:

```text
Лекция
Презентация
Методичка
Задание
Таблица
Практикум
```

```text
Краткое описание
```

Коротко объясняет, что внутри материала, для кого он и как его использовать.

Пример:

```text
Иллюстрированный ключ для лабораторных занятий по определению ископаемых двустворок.
```

```text
Курс / уровень
```

Для какого курса или уровня предназначен материал.

Примеры:

```text
Бакалавриат, 1 курс
Бакалавриат, 2 курс
Бакалавриат, 3 курс
Бакалавриат, 4 курс
Магистратура
```

```text
Год
```

Год создания или актуализации материала.

Пример:

```text
2026
```

```text
Язык
```

Язык материала.

Пример:

```text
Русский
```

```text
Теги
```

Ключевые слова.

Примеры:

```text
палеонтология
стратиграфия
двустворки
пермь
карбон
```

```text
Уровень доступа
```

Обычно для учебных материалов:

- `Открыто`, если можно всем;
- `Только студентам`, если материал учебный;
- `По запросу`, если файл нужно выдавать вручную.

### Где появится

После публикации материал появится в разделе:

```text
Студентам
```

Он будет сгруппирован по дисциплине и типу материала.

## 8. Форма "Публикация"

Выбирается во вкладке:

```text
Публикация
```

Используется для статей, монографий, тезисов, сборников и других научных работ автора.

### Файл или приложения

Можно прикрепить PDF или дополнительные файлы.

Если PDF загружается как приложение, он появится в списке прикрепленных файлов. Для публичного отображения важно заполнить поля описания публикации.

### Поля формы

```text
Название
```

Название публикации.

```text
Авторы
```

Авторы публикации.

Пример:

```text
Силантьев В. В., Снегирёв М. К.
```

```text
Год
```

Год публикации.

```text
Тип публикации
```

Тип научной работы.

Примеры:

```text
Статья
Монография
Тезисы
Глава в книге
Материалы конференции
```

```text
Журнал / издание
```

Название журнала, сборника, издательства или конференции.

```text
DOI
```

DOI публикации, если есть.

Пример:

```text
10.1234/example.2026
```

```text
Внешняя ссылка
```

Ссылка на публикацию на внешнем сайте, если есть.

Примеры:

```text
https://doi.org/...
https://elibrary.ru/...
```

```text
Краткое описание / аннотация
```

Описание публикации. Можно использовать как короткую аннотацию.

```text
Научная тема
```

Тема, к которой относится публикация.

Можно выбрать из списка или заполнить текстом, если поле такое поддерживает в текущей форме.

```text
Регион
```

Географический регион исследования.

Примеры:

```text
Среднее Поволжье
Волго-Урал
Восточно-Европейская платформа
```

```text
Геологический возраст
```

Возраст или стратиграфический интервал.

Примеры:

```text
Пермь
Карбон
Палеозой
```

```text
Язык
```

Язык публикации.

```text
Избранная публикация
```

Если включено, публикация может появиться на главной странице в блоке избранных публикаций.

```text
Теги
```

Ключевые слова публикации.

```text
Уровень доступа
```

Обычно:

- `Открыто`, если публикация публичная;
- `По запросу`, если полный текст нужно выдавать вручную;
- `Только владелец`, если это служебная запись.

### Где появится

После публикации запись появится в разделе:

```text
Публикации
```

Если включена избранность, публикация также может появиться на главной странице.

## 9. Форма "Фотография"

Выбирается во вкладке:

```text
Фотография
```

Используется для фотоархива: экспедиции, обнажения, образцы, керн, шлифы и т. д.

### Загрузка фото

Для фотографии выбирается один файл изображения.

Подходящие форматы:

- JPG;
- PNG;
- TIFF;
- другие изображения, которые поддерживает браузер.

Если название пустое, после загрузки оно может быть автоматически взято из имени файла.

### Поля формы

```text
Название
```

Название или подпись фотографии.

```text
Год
```

Год съемки.

```text
Место / локация
```

Конкретное место съемки.

Примеры:

```text
р. Сок
Печищи
Сокские карьеры
```

```text
Регион
```

Более широкий регион.

Пример:

```text
Среднее Поволжье
```

```text
Тип объекта
```

Что изображено на фотографии.

Примеры:

```text
Экспедиция
Обнажение
Керн
Образец
Ископаемое
Шлиф
```

```text
Геологический возраст
```

Возраст объекта или разреза.

```text
Описание
```

Комментарий к фотографии.

```text
Автор
```

Автор фотографии.

```text
Альбом / группа
```

Группа, в которую попадет фотография.

Примеры:

```text
Экспедиции 1983-2026
Геологические объекты
Образцы
```

```text
Правила использования
```

Комментарий о том, как можно использовать фото.

Пример:

```text
Для сайта и учебных материалов
```

```text
Связанные публикации
```

Публикации, к которым относится фотография.

```text
Связанные научные темы
```

Темы, к которым относится фотография.

```text
Теги
```

Ключевые слова для поиска и группировки.

```text
Уровень доступа
```

Обычно для фото используется `Открыто`, если оно должно быть видно в фотоархиве.

### Где появится

После публикации фотография появится в разделе:

```text
Фотоархив
```

Она также может отображаться в связанных научных темах и на карте экспедиций, если привязать ее к точке карты.

## 10. Форма "Научная тема"

Выбирается во вкладке:

```text
Научная тема
```

Научная тема объединяет публикации, фотографии и архивные материалы вокруг одного направления исследования.

### Поля формы

```text
Название
```

Название темы.

Пример:

```text
Пермские отложения Среднего Поволжья
```

```text
Регион исследования
```

Где ведется исследование.

```text
Геологический возраст
```

Возраст или период.

```text
Краткое описание
```

Короткий текст для карточки темы.

```text
Подробное описание / статья темы
```

Большой текст темы. Поле поддерживает Markdown.

Можно использовать:

```text
# Заголовок
## Подзаголовок
**жирный текст**
*курсив*
- список
1. нумерованный список
[ссылка](https://example.com)
![подпись](/uploads/image.png)
```

В редакторе можно загружать изображения. При загрузке сайт спросит, в какую папку сохранить файл.

```text
Теги
```

Ключевые слова темы.

```text
Уровень доступа
```

Обычно `Открыто`, если тема должна быть видна в разделе "Научная работа".

### Где появится

После публикации тема появится в разделе:

```text
Научная работа
```

Связи с публикациями, фото и архивом считаются по соответствующим привязкам в других разделах.

## 11. Форма "Архивный документ"

Выбирается во вкладке:

```text
Архивный документ
```

Используется для неопубликованных или служебных материалов:

- полевые дневники;
- схемы;
- таблицы;
- сканы;
- рукописи;
- рабочие документы;
- дополнительные файлы.

### Файл или приложения

Можно загрузить один или несколько файлов. Первый файл может стать основным файлом документа.

### Поля формы

```text
Название
```

Название архивного документа.

```text
Тип документа
```

Что это за документ.

Примеры:

```text
Полевой дневник
Схема
Таблица
Скан
Рукопись
Текст
```

```text
Год
```

Год создания документа или год полевых работ.

```text
Регион
```

Регион, к которому относится документ.

```text
Научная тема
```

Тема, к которой относится документ. Выбирается из уже созданных научных тем.

Если тем нет, сначала создайте тему в разделе "Научные темы".

```text
Описание документа
```

Публичное описание документа.

```text
Комментарий владельца
```

Непубличный комментарий. Используется для внутренних заметок.

Пример:

```text
Оригинал хранится в кабинете 204.
```

```text
Теги
```

Ключевые слова.

```text
Уровень доступа
```

Для архивных документов особенно важен доступ:

- `Открыто` — документ виден всем;
- `По запросу` — запись видна, но файл выдается вручную;
- `Только студентам` — доступ для учебного контекста;
- `Только владелец` — внутренний документ.

### Где появится

После публикации запись появится в разделе:

## 26. Практический порядок добавления материалов

Для учебного материала:

1. Открыть "Быстрое добавление".
2. Выбрать "Учебный материал".
3. Загрузить файл.
4. Выбрать папку `lectures` или другую подходящую.
5. Заполнить название.
6. Заполнить дисциплину.
7. Заполнить тип материала.
8. Заполнить описание.
9. Указать курс, год, язык.
10. Добавить теги.
11. Выбрать доступ.
12. Нажать "Опубликовать".

Для публикации:

1. Открыть "Быстрое добавление".
2. Выбрать "Публикация".
3. Заполнить название, авторов, год и тип.
4. Указать журнал / издание.
5. Добавить DOI или внешнюю ссылку, если есть.
6. Заполнить аннотацию.
7. Указать тему, регион и возраст.
8. Прикрепить PDF или дополнительные файлы.
9. При необходимости включить "Избранная публикация".
10. Нажать "Опубликовать".

Для фотографии:

1. Открыть "Фотографии".
2. Нажать "Добавить фото".
3. Загрузить изображение.
4. Заполнить название.
5. Указать год, регион, тип объекта и возраст.
6. Заполнить описание и автора.
7. Выбрать альбом / группу.
8. Привязать публикации или темы, если нужно.
9. Нажать "Опубликовать".

Для научной темы:

1. Открыть "Научные темы".
2. Нажать "Новая тема".
3. Заполнить название.
4. Указать регион и возраст.
5. Написать краткое описание.
6. Заполнить подробное описание.
7. Добавить теги.
8. Нажать "Опубликовать".

Для архивного документа:

1. Открыть "Архивные материалы".
2. Нажать "Добавить документ".
3. Загрузить файл.
4. Заполнить название.
5. Указать тип документа и год.
6. Указать регион.
7. Выбрать научную тему.
8. Заполнить публичное описание.
9. Заполнить внутренний комментарий, если нужно.
10. Выбрать доступ.
11. Нажать "Опубликовать".

Для точки на карте:

1. Открыть "Карта экспедиций".
2. Нажать "Новое место".
3. Кликнуть по карте.
4. Заполнить название, регион, годы и описание.
5. Загрузить обложку, если нужна.
6. Выбрать связанные фото, публикации, материалы, темы и архивные документы.
7. Нажать "Сохранить".

3. Скачайте резервную копию.
4. Только после этого удаляйте много записей или чистите файлы.

- удалением архивных документов;
- переносом сайта на сервер.

## 28. Частые ошибки

### Материал не появился на сайте

Проверьте:

- нажали ли "Опубликовать", а не только "Сохранить черновик";
- не стоит ли доступ "Только владелец";
- не скрыт ли материал;
- правильно ли выбран раздел.

### Файл загрузился, но не виден в записи

Проверьте:

- прикреплен ли файл в списке приложений;
- выбран ли основной файл;
- не удалена ли привязка крестиком;
- не удален ли файл в файловом менеджере.

### Фото не отображается

Проверьте:

- есть ли путь вида `/uploads/...`;
- файл действительно есть в файловом менеджере;
- это изображение, которое поддерживает браузер;
- путь не содержит ошибок.

### На карте нет точки

Проверьте:

- сохранено ли место;
- указаны ли корректные `lat` и `lng`;
- есть ли название места;
- не удалена ли точка.

### Непонятно, куда сохранился файл

Откройте:

```text
Файловый менеджер
```

и найдите файл по имени или по папке.

## 29. Минимальная безопасная схема работы

Самая спокойная схема:

1. Сначала загрузить файлы.
2. Заполнить форму.
3. Сохранить черновик.
4. Проверить, как выглядит превью.
5. Нажать "Опубликовать".
6. Открыть публичный сайт и проверить результат.

Так меньше шанс случайно опубликовать неполную запись.
