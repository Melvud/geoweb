
Центральная идея технического задания
Нужно разработать персональный научно-образовательный сайт профессора-геолога.
Главный объект разработки — админка, через которую владелец сайта сможет самостоятельно добавлять, редактировать, скрывать, публиковать и систематизировать материалы.
Публичная часть сайта должна автоматически формироваться на основе данных, загруженных через админку.
1. Главная задача админки
Админка должна позволять владельцу сайта самостоятельно управлять следующими типами материалов:
1. Учебные материалы2. Публикации3. Фотографии4. Научные темы5. Архивные материалы6. Страницы сайта
То есть сайт должен работать не как набор вручную сверстанных страниц, а как управляемое хранилище материалов.

3. Что должна уметь админка
Минимальный набор функций:
создавать новый материал;редактировать материал;загружать файлы;загружать изображения;добавлять теги;выбирать раздел сайта;назначать уровень доступа;публиковать материал;скрывать материал;удалять материал;сохранять черновик;искать материалы внутри админки;фильтровать материалы по типу, теме, году, региону и статусу.

4. Публичная часть сайта
Публичная часть должна брать данные из админки и показывать их пользователю.
Главное меню:
ГлавнаяОбо мнеСтудентамНаучная работаПубликацииФотоархивАрхив материаловКонтакты
Но важно: эти страницы не должны вручную заполняться разработчиком каждый раз. Владелец добавляете материал в админке — он появляется в нужном разделе сайта.
Например:
Вы добавили презентацию по палеонтологии.Выбрали дисциплину: Палеонтология.Выбрали тип: Презентация.Поставили доступ: Открыто.После сохранения материал автоматически появился в разделе:Студентам → Палеонтология → Презентации.

6. Минимальная первая версия
Для первой версии сайта достаточно сделать:
1. Публичный сайт с основными разделами2. Админку3. Добавление учебных материалов4. Добавление публикаций5. Добавление фотографий6. Добавление научных тем7. Загрузку файлов и изображений8. Поиск9. Простые фильтры10. Уровни доступа: открыто / скрыто / по ссылке
Можно отложить на вторую очередь:
личный кабинет студентов;сложную систему прав доступа;интерактивную карту экспедиций;английскую версию;массовую загрузку большого фотоархива;Telegram-бот;каталог палеонтологических объектов с расширенной таксономией.

Краткая формулировка для разработчика
Центральная задача проекта — разработать не просто сайт, а админку для управления научными, учебными и архивными материалами. Через админку владелец сайта должен самостоятельно добавлять публикации, лекции, презентации, фотографии, архивные документы и научные темы. Публичная часть сайта должна автоматически формироваться из этих материалов. В первой версии нужны основные разделы сайта, карточки материалов, загрузка файлов, теги, поиск, фильтры и уровни доступа.
Короче: Главная ценность сайта — удобная админка и правильно организованная база материалов. Дизайн важен, но вторичен.
